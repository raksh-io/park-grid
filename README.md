# 🅿️ ParkGrid - Smart Parking Solutions

ParkGrid is a modern, real-time smart parking management system designed to eliminate the hassle of finding and managing parking spaces. Built with a mobile-first approach, it offers a seamless experience for both users and parking administrators.

![ParkGrid Logo](logo.png)

## 🚀 Features

### For Users
- **Real-Time Availability**: Instantly see which parking slots are free (Green) or occupied (Red).
- **Instant Booking**: Secure your spot in seconds by selecting a duration and vehicle type.
- **Secure Payments**: Integrated **Razorpay** gateway for safe and quick transactions.
- **Overstay Management**: Automated fine calculation for users who exceed their booking time.
- **Dashboard Notifications**: Real-time alerts when your booking is about to expire.

### For Admins
- **Admin Dashboard**: Comprehensive overview of all parking slots and their current status.
- **Slot Management**: Add, edit, or delete parking slots directly from the interface.
- **Active Booking Details**: View detailed information about current occupants (Vehicle #, User Email, etc.).
- **Manual Release**: Ability to manually free up slots in case of emergencies or maintenance.
- **CSV Reports**: Export active booking data for record-keeping and analysis.

## 🛠️ Tech Stack

- **Frontend**: Vanilla HTML5, CSS3 (Modern Glassmorphism Design), Vanilla JavaScript (ES6+).
- **Backend**: [Supabase](https://supabase.com/) (Real-time Database & Authentication).
- **Payments**: [Razorpay](https://razorpay.com/) Checkout Integration.
- **Icons/Fonts**: Google Fonts (Inter), Emojis for intuitive UI.

## ⚙️ Setup & Installation

### 1. Database Setup (Supabase)
Create a new project on Supabase and run the following SQL in the **SQL Editor** to create the required table:

```sql
-- Create the slots table
CREATE TABLE public.slots (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  slot TEXT NOT NULL UNIQUE,
  status TEXT DEFAULT 'available', -- 'available' or 'booked'
  vehicle TEXT,
  vehicle_type TEXT,
  start_time TIMESTAMPTZ,
  end_time TIMESTAMPTZ,
  user_email TEXT,
  fine_amount NUMERIC DEFAULT 0,
  fine_start_time TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Real-time for the slots table
ALTER PUBLICATION supabase_realtime ADD TABLE public.slots;
```

### 2. Authentication Setup
In your Supabase project:
1. Go to **Authentication** > **Providers**.
2. Enable **Email/Password**.
3. (Optional) Enable **Google OAuth** and provide your credentials.

### 3. Configuration
Update `config.js` with your Supabase credentials:

```javascript
const SUPABASE_URL = "YOUR_SUPABASE_URL";
const SUPABASE_ANON_KEY = "YOUR_SUPABASE_ANON_KEY";
```

### 4. Razorpay Integration
Currently, the project uses a test key in `payment.html`. For production:
1. Sign up on [Razorpay](https://razorpay.com/).
2. Replace the `key` in `payment.html` with your **Live Key ID**.

## 🌐 Deployment

Since ParkGrid is a static web application, you can deploy it for free using various platforms:

### Vercel / Netlify
1. Connect your GitHub repository.
2. **Configure Environment Variables**: Go to Project Settings > Environment Variables and add:
   - `SUPABASE_URL`: Your Supabase Project URL
   - `SUPABASE_ANON_KEY`: Your Supabase Anon Key
   - `RAZORPAY_KEY`: Your Razorpay Test/Live Key
3. Deploy! The build script will automatically inject these into your code.


### GitHub Pages
1. Go to Repository **Settings** > **Pages**.
2. Select the `main` branch and `/root` folder.
3. Your site will be live at `https://<username>.github.io/park-grid/`.
